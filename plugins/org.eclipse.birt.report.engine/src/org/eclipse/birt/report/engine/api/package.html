This is a Tool-writer's interface for using the BIRT report engine. To use the engine, first create an engine configuration (EngineConfig) object. The report engine (ReportEngine) constructor takes the configuration object to initialize the report engine.

A report engine supports running several types of task. Examples are RunReportTask, RunDataSetTask, GetDnamicParameterListTask, etc. To run a report, the following steps may be involved:

  • The report design is opened by calling the openReportDesign() method on ReportEngine. The method returns a ReportRunnable object.
  • From the ReportRunnable object, parameter definitions are retrieved. A UI page is constructed based on the definitions to prompt for parameter values.
  • A RunReportTask is constructed, and report parameters are set.
  • The run() method on RunReportTask is called.
Code segments are shown below to demonstrate how to use the engine.

Simplest use of report engine

To get a report to run, one needs only 7 lines of code:

ReportEngine engine = new ReportEngine(null);
IReportRunnable design = engine.openReportDesign("C:/temp/test.design");
IRunAndRenderTask task = engine.createRunAndRenderTask(design);
IOutputSetting setting = new OutputSetting();
setting.setOutputFileName("C:/temp/test.html");
task.setOutputSetting(setting);
task.run();

No customization is done so engine will asserts its default behavior: output the report to HTML format at C:/temp/test.html, assuming JVM locale. The report design can not contain images, charts ot hyperlinks, or otherwise engine has to be configured with corresponging handlers.

Using engine with heavy customization

The following example customizes the engine in detail:

// Engine configuration
EngineConfig config = new EngineConfig();
config.setBIRTHome("C:/birt/"); // Configuring where BIRT engine is installed
config.setDataDriverDir("C:/drivers/"); // Configures where data drivers are stored.
config.setImageHandler(IImage.DESIGN_IMAGE, designImageHandler); // customize design time image handler
config.setImageHandler(IImage.CUSTOM_IMAGE, customImageHandler); // configure customized image (i.e., chart) handler
config.setImageHandler(IImage.CUSTOM_IMAGE, customImageHandler); // configure customized image (i.e., chart) handler
config.addScriptableJavaObject("myJavaObject", aJavaObject); // Customized Java object to scripting

// Create engine and open report design
ReportEngine engine = new ReportEngine(config); //Create engine with configuration
IReportRunnable design = engine.openReportDesign("C:/temp/test.design");

// Get parameter definitions
Collection parameters = design.getParameterDefns(false, myLocale); // get parameter definitions, with display names, default values, etc. in myLocale
// Present parameter prompt page and receive inputs. Use class ParameterMapBuilder to build up a parameter value colleciton

// Create task to run and render the report
IRunAndRenderTask task = engine.createRunAndRenderTask(design);

// output options
XHTMLOutputSetting setting = new XHTMLOutputSetting(); // assume this is a third-party format
setting.setOutputFileName("C:/temp/test.html");
setting.setOutputFormat("html"); // XHTML is still "html" with certain changes
setting.setEmbeddable(true); // XHTML also supports embeddable
setting.setLocale(myLocale);
setting.setExtensionID("org.mycompany.report.extension.html"); // Now 2 emitters support html format, choose which one to use
task.setOutputSetting(setting);

task.run();
